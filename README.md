# 🧘 Local Yoga Calendar

A unified class calendar that aggregates schedules from local yoga studios into one beautiful view, hosted for free on GitHub Pages.

## How It Works

```
GitHub Actions (runs nightly)
  └── yoga_scraper.py runs
       └── scrapes studio websites
            └── saves to schedule_data.json
                  └── GitHub Pages serves index.html
                        └── index.html reads schedule_data.json → shows calendar
```

Think of it as a newspaper. Python is the journalist who gathers information each night. `schedule_data.json` is the printed paper. `index.html` is the newsstand where people read it.

## Setup Guide

### Step 1: Create your GitHub repo

1. Go to [github.com](https://github.com) and create a new **public** repository
2. Name it anything — e.g. `yoga-calendar`
3. Push all these files to it:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git remote add origin https://github.com/YOUR_USERNAME/yoga-calendar.git
   git push -u origin main
   ```

### Step 2: Enable GitHub Pages

1. Go to your repo → **Settings** → **Pages**
2. Under "Source", select **Deploy from a branch**
3. Choose **main** branch, **/ (root)** folder
4. Click **Save**
5. Your site will be live at: `https://YOUR_USERNAME.github.io/yoga-calendar`

### Step 3: Configure your studios

Open `studios_config.json` and add the studios you want to scrape:

```json
{
  "studios": [
    {
      "name": "My Local Studio",
      "url": "https://www.myLocalStudio.com/schedule",
      "color": "#c17a5c",
      "enabled": true
    }
  ]
}
```

**Colors:** Each studio gets its own color on the calendar. Hex codes work great.

### Step 4: Write a scraper for each studio

This is the custom part — every website is different. Open `yoga_scraper.py` and:

1. Find the section labeled `# ── Example Studio Scrapers`
2. Create a new class by copying `ExampleStudioScraper`
3. Rename it, e.g. `MyLocalStudioScraper`
4. In the `scrape()` method, write code to find class data in the HTML

**How to inspect a studio's HTML:**
1. Open their schedule page in Chrome
2. Right-click a class name → **Inspect**
3. Look at the surrounding HTML — find what element wraps each class
4. Use `soup.find_all("div", class_="that-class-name")` to grab them all

5. Register your scraper in the `SCRAPERS` dict at the bottom of the file:
   ```python
   SCRAPERS = {
       "My Local Studio": MyLocalStudioScraper,
   }
   ```

### Step 5: Test your scraper locally

```bash
pip install httpx beautifulsoup4
python yoga_scraper.py
```

Check `schedule_data.json` — if it has class data, it worked! Then open `index.html` in your browser to preview the calendar.

### Step 6: Push and let GitHub Actions take over

```bash
git add .
git commit -m "Add studio scrapers"
git push
```

GitHub Actions (configured in `.github/workflows/scrape.yml`) will now run your scraper **automatically every night at 5am UTC** and update the live site.

You can also trigger it manually: go to your repo → **Actions** tab → **Scrape Yoga Schedules** → **Run workflow**.

## Adding a New Studio (Quick Reference)

1. Add an entry to `studios_config.json`
2. Create a new class in `yoga_scraper.py` that extends `StudioScraper`
3. Implement the `scrape()` method
4. Add it to the `SCRAPERS` dict
5. Test locally: `python yoga_scraper.py`
6. Push to GitHub

## Troubleshooting

**The calendar is empty**
- Make sure `schedule_data.json` has a `"classes"` array with items
- Open browser DevTools (F12) → Console tab for error messages

**Scraper returns no classes**
- The studio may have changed their HTML structure — re-inspect and update
- Some sites block scrapers; try adding a `User-Agent` header in `fetch_page()`
- Many studios use JavaScript to render their schedule, which plain `httpx` can't handle. In that case, look into `playwright` for Python (browser automation)

**GitHub Actions isn't running**
- Check the **Actions** tab in your repo for error logs
- Make sure your repo is public (Actions are free for public repos)
